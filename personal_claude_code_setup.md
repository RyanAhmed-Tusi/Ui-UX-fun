# Your Personal Claude Code Setup
### A Complete System for Building High-End Websites — From Brief to Deployment

---

## What This Setup Is

This is a ready-to-use personal system that:
- Gives Claude Code persistent context about your project across every session
- Defines the folder structure, commands, and workflow for every project type
- Integrates the right MCPs so Claude can reach into Figma, your browser, and GitHub
- Separates clearly what you do manually vs. what Claude handles
- Gives you repeatable slash commands for every phase of work

Set this up once per project. The investment is 30–60 minutes. The return is every Claude Code session being immediately productive with zero repeated context-setting.

---

## Part 1 — The Four Files That Power Everything

These four files live in every project. Together they are Claude's persistent memory for the project.

---

### File 1: `CLAUDE.md` — The Project Brain

This file lives at the root of your project. Claude Code reads it automatically at the start of every session. It is Claude's permanent memory for this project — commands, architecture, conventions, and constraints.

**Critical constraint:** Keep it under 200 lines. Every line consumes context budget. Frontier models reliably follow only about 150–200 instructions total, and Claude Code's own system prompt uses roughly 50 of them. Aim for under 150 lines. Short and command-first. Reference other files (design.md, motion.md) rather than pasting content here.

**Template — copy and fill in for each project:**

```markdown
# [Project Name] — Claude Code Context

## Project Overview
[2–3 sentences: what this site is, who it's for, the tone/feel]

## Tech Stack
- Framework: Next.js 14 (App Router)
- Styling: Tailwind CSS v4
- Animation: GSAP + ScrollTrigger, Motion (Framer Motion)
- 3D: React Three Fiber + @react-three/drei + @react-three/postprocessing
- Smooth Scroll: Lenis
- Package manager: pnpm

## Commands
- Dev server: `pnpm dev`
- Build: `pnpm build`
- Type check: `pnpm tsc --noEmit`
- Lint: `pnpm lint`

## Architecture
- Pages live in /src/app (App Router)
- Reusable UI components in /src/components/ui
- Page sections in /src/components/sections
- Three.js/R3F components in /src/components/three
- Custom hooks in /src/hooks
- See /src/styles/globals.css for CSS custom properties

## Design System
See design.md for all design tokens. Always pull from design.md 
for colors, typography, and spacing. Never hardcode hex values inline.

## Animation Rules
See motion.md for animation spec.
- All animations must respect prefers-reduced-motion
- Only animate GPU-accelerated properties: transform, opacity, filter
- Scroll animations use GSAP ScrollTrigger. UI animations use Motion.
- Lenis smooth scroll is always active. Connect ScrollTrigger to Lenis.

## Asset Locations
- /public/models/ — .glb 3D models (Draco-compressed)
- /public/frames/ — image sequence WebP frames
- /public/lottie/ — .json Lottie animation files
- /public/hdri/ — .hdr environment maps for 3D lighting
- /public/fonts/ — .woff2 font files (self-hosted)

## Code Conventions
- TypeScript everywhere. No implicit any.
- Functional components with hooks. No class components.
- Dynamic imports for Three.js components (SSR safety)
- All 3D canvas must have `ssr: false` in dynamic()
- Use CSS custom properties (from globals.css) for theming

## Performance Targets
- LCP under 2.5s on fast 3G
- 60fps on mid-range device for all animations
- 3D models: under 5MB total, under 100K triangles
- Image sequences: WebP, ~10-20KB per frame, 240–300 frames max
```

### File 3: `motion.md` — The Animation Spec

This defines what animates, when, and how. Claude reads this to generate animation code that matches your intended motion design rather than making guesses.

**Template:**

```markdown
# Motion Design Spec — [Project Name]

## Scroll Library
Lenis smooth scroll (duration: 1.2, easing: expo-out)
GSAP ScrollTrigger connected to Lenis RAF loop.

## Global Animation Principles
- Prefer ease-out for entrances (fast arrival, gentle settle)
- Prefer ease-in for exits (slow start, fast departure)
- Use spring physics (back.out, elastic.out) sparingly — only for playful moments
- All text reveals use SplitText by characters or words
- No element should animate more than 80px of Y travel on entrance
- Stagger: 60ms between sibling elements, 120ms between parent/child groups

## Timing Reference
Micro-interactions (hover, click):   150ms ease-out
UI transitions (modal, dropdown):    280ms ease-out
Entrance animations:                 600ms power3.out
Section transitions:                 800ms expo.out
Cinematic/scroll moments:            scrub: 1 (GSAP scrub value)

## Entrance Patterns (use these named patterns)
- fade-up:     from { opacity: 0, y: 40 } → ease: power3.out, 600ms
- fade-in:     from { opacity: 0 }         → ease: power2.out, 400ms
- slide-left:  from { opacity: 0, x: 60 } → ease: power3.out, 700ms
- scale-in:    from { opacity: 0, scale: 0.92 } → ease: back.out(1.5), 600ms
- chars-rise:  SplitText chars, from { opacity: 0, y: 60, rotateX: -90 }, stagger: 0.03, 700ms

## Scroll Behavior Patterns
Hero section:        No scroll animation — static, loads with entrance animation
Story sections:      Pin with ScrollTrigger, scrub-driven content reveals
Image sequence:      Pin 3000px, scrub: true, canvas frame = scroll progress
3D model section:    Pin 4000px, camera orbits model based on scroll.offset
Feature sections:    Trigger-on-enter, not scrub. Elements enter as you reach them.
Footer:              Fade-in, no scroll pin

## 3D Animation Values
Jet model scroll:
  scroll 0%   → camera: front view (0, 0, 5)
  scroll 33%  → camera: side view (5, 1, 2), rotation.y += 90deg
  scroll 66%  → camera: rear-quarter (3, 2, -3)
  scroll 100% → camera: top-down (0, 6, 0)

## Hover States
Cards: translateY(-8px), shadow increase, 200ms ease-out
Buttons: scale(1.03), 150ms ease-out
Links: color to accent, underline reveal, 150ms ease-out
3D element: OrbitControls activate on hover, deactivate on scroll

## Accessibility
All animations wrapped in:
  @media (prefers-reduced-motion: reduce) → duration: 0.01ms, no scrub
```

## Part 3 — Custom Slash Commands

Files in `.claude/commands/` become slash commands. Type `/project:new-section` in Claude Code and it runs that command automatically. These are your project's repeatable workflows.

---

**`.claude/commands/new-section.md`**

```markdown
Build a new page section with the following brief: $ARGUMENTS

Steps:
1. Read design.md for all color, typography, and spacing tokens
2. Read motion.md for the correct animation pattern for this section type
3. Create the component in /src/components/sections/[name].tsx
4. Use TypeScript. Use CSS custom properties from design.md, not hardcoded values.
5. Add entrance animation using GSAP ScrollTrigger trigger-on-enter pattern
6. Wrap animations in prefers-reduced-motion check
7. Export and add to the appropriate page in /src/app/
8. Show me the component code and confirm which page it's added to.
```

**`.claude/commands/build-3d.md`**

```markdown
Set up a Three.js scene with the following spec: $ARGUMENTS

Steps:
1. Check if R3F is installed. If not, install: pnpm add @react-three/fiber @react-three/drei @react-three/postprocessing three
2. Create the component in /src/components/three/[name].tsx
3. Dynamic import with ssr: false in the page that uses it
4. Read motion.md for scroll animation values
5. Connect to GSAP ScrollTrigger via useScroll from drei or scroll.offset
6. Add environment lighting from /public/hdri/ (useEnvironment from drei)
7. Add postprocessing (Bloom at minimum for emissive surfaces)
8. Add OrbitControls with damping, limit polar angle
9. Handle resize and pixel ratio properly
10. Mobile: reduce model detail or substitute a static fallback
11. Show me the complete component.
```

**`.claude/commands/build-sequence.md`**

```markdown
Build a canvas image sequence section. $ARGUMENTS

Steps:
1. Frames are at /public/frames/[folder-name]/frame_XXXX.webp
2. Count available frames (check the folder)
3. Create CanvasSequence component in /src/components/three/CanvasSequence.tsx
4. Preload all frames as Image objects before the section becomes visible
5. Pin the section with ScrollTrigger for [described scroll height]
6. Map scroll progress 0→1 to frame index 0→totalFrames
7. Draw the current frame on canvas using requestAnimationFrame
8. Add crossfade between frames if needed for smoothness
9. Handle window resize (recalculate canvas dimensions)
10. Show loading progress while frames preload
11. Add any text overlay animations from motion.md
```

**`.claude/commands/review.md`**

```markdown
Review the current state of the project.

1. Check all TypeScript errors: run `pnpm tsc --noEmit` and report issues
2. Check for console errors (if browser MCP is available, open the dev server and check)
3. Review animation code — are all animations using GPU-accelerated properties only?
4. Check that all images in /public/img/ are WebP format
5. Check that all 3D models in /public/models/ are Draco-compressed (.glb)
6. Verify prefers-reduced-motion is handled in all animation hooks
7. Check for any hardcoded color values not from design.md tokens
8. List any unfinished TODO comments in the codebase
9. Report a summary of findings and prioritized issues.
```

**`.claude/commands/optimize.md`**

```markdown
Optimize performance for the following: $ARGUMENTS

Check and fix:
1. Image loading: are all images using Next.js <Image> component?
2. 3D model: is it lazily loaded with Suspense?
3. Image sequence: is preloading happening before the section is visible?
4. Code splitting: are Three.js components dynamically imported?
5. Font loading: are fonts preloaded in layout.tsx <head>?
6. Animation: run a check for any non-GPU-accelerated properties being animated
7. Bundle size: check if any libraries could be imported more specifically
8. Run Lighthouse suggestions if browser MCP is available
9. Fix identified issues and report what was changed.
```

**`.claude/commands/commit.md`**

```markdown
Commit the current work to GitHub.

1. Run `pnpm tsc --noEmit` — only proceed if no errors
2. Run `pnpm build` — only proceed if build succeeds  
3. Stage all changed files
4. Write a descriptive commit message based on what was built/changed
5. Commit and push to the current branch
6. Report the commit hash and what was included.
```

## Part 4 — MCP Setup

Run these once to configure MCPs for Claude Code. They persist across all projects.

```bash
# Figma MCP — design-to-code bridge
# First: install Figma's MCP server from figma.com/developers/mcp
# Then in Claude Code:
claude mcp add figma npx @figma/mcp-server

# Browser MCP — Claude can open, screenshot, and test your site
claude mcp add puppeteer npx @modelcontextprotocol/server-puppeteer

# GitHub MCP — Claude commits and manages branches
claude mcp add github npx @modelcontextprotocol/server-github

# Motion AI Kit MCP — correct Motion/Framer Motion animation code
# Requires Motion+ subscription at motion.dev
# Install via: npx motion-ai-kit install
# Then follow prompts to select Claude Code
```

**Verifying MCPs are active:**
In Claude Code, type `/mcp` to see all connected servers. Green = connected.

## Part 5 — The Session Workflow

A repeatable process for every working session with Claude Code.

### Starting a Session

Always start with context. Don't jump straight to building.

```
1. Open Claude Code in your project directory
   (CLAUDE.md is auto-loaded)

2. Brief Claude on today's goal:
   "Today we're building the 3D jet section. 
   The model is at /public/models/jet.glb. 
   Refer to motion.md for the scroll animation values.
   The section should pin for 4000px and rotate the camera 
   around the jet from front to rear quarter."

3. Use Plan Mode for anything complex:
   Type /plan before your request for substantial features.
   Claude proposes an approach, you review it, then approve execution.
   This catches misunderstandings before 30 minutes of wrong code is written.

4. Claude builds. You review.
```

### Mid-Session Pattern

```
Build one thing at a time. Don't ask Claude to build 
the entire site in one session.

Good: "Build the hero section with the entrance animation"
Bad:  "Build all 8 sections of the homepage"

After Claude builds something:
- Open the browser (or Claude opens it via Puppeteer MCP)
- Review visually
- Note specific issues: "The stagger is too fast" not "it looks wrong"
- Claude iterates
- When it looks right, run /project:commit
```

### Ending a Session

```
Before ending:
1. Run /project:review — catch issues before next session
2. Run /project:commit — save all work to GitHub
3. Note what's next in a comment or in the project's README
   so the next session starts with clear direction
```

## Part 6 — Phase-by-Phase Manual vs. Automated Reference

| Phase | You Do | Claude Does |
|---|---|---|
| Discovery | Brief writing, creative vision, final IA approval | Research, competitor analysis, IA structure generation |
| UX Design | Flow decisions, approval of structure | v0 component generation, layout scaffolding |
| Visual Design | Art direction, font selection, Figma design decisions | design.md from Stitch, Figma MCP implementation |

| Video → Frames | Running FFmpeg, reviewing output quality | FFmpeg command generation, canvas implementation |
| 3D Models | Meshy prompting, Blender cleanup (20%), quality review | All R3F code, scene setup, scroll connection |
| Lottie / Rive | Design in Rive.app, review output | Integration code, trigger/control logic |
| All Animation Code | Review, easing feedback, approval | 100% of GSAP, Motion, ScrollTrigger code |
| Development | Review, bug reporting, UX feedback | 100% of React, TypeScript, Next.js code |
| Performance | Real device testing, final quality gate | Optimization suggestions + implementation |
| Deployment | Approve merge to main | Git workflow, Vercel config, CI setup |

## Part 7 — Starting a New Project (Checklist)

Copy this for every new project:

```
□ Create project folder and init Next.js:
  pnpm create next-app@latest [project-name] --typescript --tailwind --app

□ Install core dependencies:
  pnpm add gsap @gsap/react
  pnpm add motion
  pnpm add @studio-freight/lenis
  pnpm add @react-three/fiber @react-three/drei @react-three/postprocessing three
  pnpm add @lottiefiles/react-lottie-player
  pnpm add @splinetool/react-spline (if using Spline)

□ Copy CLAUDE.md template → fill in project details
□ Generate design.md from Google Stitch or fill template
□ Fill motion.md template with project animation spec
□ Create assets-manifest.md (empty, fill as assets arrive)
□ Create .claude/commands/ folder and copy all command files
□ Connect MCPs (figma, puppeteer, github, motion-ai-kit)
□ Create GitHub repo, connect to Vercel for auto-deploy
□ Add fonts to /public/fonts/ and preload in layout.tsx
□ Set up CSS custom properties in globals.css from design.md

□ First Claude Code session:
  "Project is set up. CLAUDE.md, design.md, and motion.md are ready.
  Start by building the base layout: navbar, page wrapper with Lenis,
  GSAP registered with ScrollTrigger connected to Lenis,
  and the footer. Then we'll build section by section."
```

## Part 8 — Quick Reference: What to Say to Claude Code

**Starting a section:**
"Build the [section name] section. Refer to design.md for tokens. 
The animation should use the [pattern from motion.md] entrance. 
The content is: [describe what text, images, elements are in it]."

**Connecting scroll to 3D:**
"Connect GSAP ScrollTrigger to the jet model. 
Refer to motion.md for the camera position values at each scroll milestone. 
Pin the section for 4000px. Scrub the camera along the path."

**Image sequence:**
"Build the canvas image sequence. Frames are at /public/frames/window/. 
Count the frames. Pin the section for 3500px. Map scroll to frames. 
Add a 'SCROLL' label that fades out after the first 10% of scroll."

**Fixing something specific:**
"The stagger on the feature cards is too fast. Change it from 60ms to 120ms. 
Also, the entrance Y travel is too much — reduce from 60px to 30px."

**Performance:**
"Run a performance review. Check for any animations not using 
GPU-accelerated properties. Confirm 3D model is lazy-loaded. 
Fix what you find."

**Shipping:**
"/project:commit — we finished the hero and story sections today."
